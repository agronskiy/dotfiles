# CLAUDE.md Management

The user's global CLAUDE.md is assembled from two source files, merged by `claude-setup`:

| Source | Path | Visibility |
|--------|------|------------|
| Public | `~/.dotfiles/.claude/CLAUDE.md` | Tracked in public dotfiles repo |
| Local  | `~/.dotfiles-local/.claude/CLAUDE.md` | Tracked in private dotfiles repo |

The merged output lives at `~/.claude/CLAUDE.md` (a regular file, not a symlink).

## IMPORTANT: Never edit ~/.claude/CLAUDE.md directly

It is auto-generated by `claude-setup`. Always edit the source files:

- **Public preferences** (coding style, communication, tool usage): edit `~/.dotfiles/.claude/CLAUDE.md`
- **Work/private context** (internal URLs, team conventions, secrets): edit `~/.dotfiles-local/.claude/CLAUDE.md`

After editing, run `claude-setup` to regenerate the merged file.

## IMPORTANT: Confirm visibility before editing

Before making any change, always ask the user explicitly:

> "Should this go in the **public** (`~/.dotfiles`) or **local** (`~/.dotfiles-local`) CLAUDE.md?"

- Public → edit `~/.dotfiles/.claude/CLAUDE.md`
- Local → edit `~/.dotfiles-local/.claude/CLAUDE.md`

Do not assume — always confirm.

## How to decide public vs. local

- If it would be fine in a public GitHub repo → public
- If it references internal tools, teams, or private context → local

See `references/structure.md` for details on the merge process.
